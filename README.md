# 顧客課題PDF作成Webアプリ

パーソナルトレーナーが、顧客ごとの次回トレーニングまでの課題を作成し、LINEなどで共有しやすいPDFとして出力するためのMVPアプリです。

顧客情報、書籍情報、課題プラン、テンプレートはブラウザの `localStorage` に保存します。バックエンドなしで試せる構成ですが、OpenAI APIキーを設定するとサーバー側API Route経由でAI課題案を生成できます。APIキーがない場合でも、既存のモック生成で動作します。

## 主な機能

- 顧客情報の登録・編集・削除
  - 目標、体型・悩み、生活習慣、睡眠、食事、運動歴、器具、怪我・痛み、トレーナーメモを管理
- 課題プラン作成
  - 顧客、期間、PDF出力モード、日別課題を管理
  - 日付範囲から曜日付きの日別課題を自動生成
- AI課題案生成
  - OpenAI APIキーあり: OpenAI Responses APIで日別課題案を生成
  - OpenAI APIキーなし: ルールベースのモック生成へフォールバック
  - 生成後もすべて手動修正可能
- 書籍学習課題
  - 書籍と章・節・ページ範囲を登録
  - 「4日進む・2日復習」のようなペースで日別学習課題へ自動割り当て
- テンプレート管理
  - よく使う課題文を登録し、日別課題へ反映
- PDFプレビュー・ダウンロード
  - A4縦の課題PDFをプレビューし、ダウンロード可能
  - スマホで見やすいように要点を短く表示
- localStorage保存
  - 再読み込み後も同じブラウザではデータを復元
  - zodで保存データを検証

## セットアップ方法

### 1. リポジトリを取得

```bash
git clone <repository-url>
cd <repository-directory>
```

### 2. 依存関係をインストール

```bash
npm install
```

### 3. 環境変数を設定（任意）

OpenAI APIで課題案を生成したい場合は、プロジェクトルートに `.env.local` を作成します。

```bash
OPENAI_API_KEY=sk-...
# 任意: 未指定時は gpt-5-mini を使用
OPENAI_ASSIGNMENT_MODEL=gpt-5-mini
```

`OPENAI_API_KEY` はサーバー側の `src/app/api/generate-assignment/route.ts` でのみ読み取ります。クライアント側には公開しません。

## 起動方法

開発サーバーを起動します。

```bash
npm run dev
```

起動後、ブラウザで以下を開きます。

```text
http://localhost:3000
```

## 環境変数の設定方法

| 変数名 | 必須 | 説明 |
| --- | --- | --- |
| `OPENAI_API_KEY` | 任意 | OpenAI APIで課題案を生成する場合に設定します。未設定の場合はモック生成を使います。 |
| `OPENAI_ASSIGNMENT_MODEL` | 任意 | AI課題案生成に使うモデル名です。未設定時は `gpt-5-mini` を使います。 |

`.env.local` はGitにコミットしないでください。`.gitignore` で `.env*` を除外しています。

## OpenAI APIキーなしでモック生成する方法

`OPENAI_API_KEY` を設定しない状態でもアプリは動作します。

1. `/customers` で顧客を登録します。
2. `/assignments` で顧客を選択し、課題期間を設定します。
3. 日別課題が生成されていることを確認します。
4. 「AIで課題案を作成」を押すと、API Route側でAPIキー未設定を検知し、モック生成へ自動フォールバックします。
5. OpenAI APIでエラーが出た場合や明示的にモックを使いたい場合は、「モックで作成」を押します。

モック生成は、顧客の目標、運動歴、器具、睡眠、食事、怪我・痛みなどをもとに簡易ルールで課題案を作ります。

## OpenAI APIキーありでAI生成する方法

1. `.env.local` に `OPENAI_API_KEY` を設定します。
2. 開発サーバーを再起動します。
3. `/assignments` で顧客と課題期間を設定します。
4. 必要に応じて書籍学習ルールや休養日を設定します。
5. 「AIで課題案を作成」を押します。
6. API RouteがOpenAI Responses APIを呼び出し、構造化JSONで日別課題案を受け取ります。
7. 生成結果はzod schemaで検証され、検証に成功した場合だけ日別課題へ反映されます。
8. 内容を確認し、必要に応じて手動で修正します。

AIには、医療的な診断や治療表現を避けること、痛みが出たら中止すること、不安があればトレーナーに相談すること、書籍本文を出さず範囲だけを表示することを指示しています。

## 顧客登録の使い方

1. ナビゲーションから「顧客管理」を開きます。
2. 「＋ 顧客を追加」を押します。
3. 顧客名と目標を入力します。
4. 必要に応じて以下を入力します。
   - 年齢、性別、身長、現在体重、目標体重
   - 体型・悩み、生活習慣、睡眠、食事習慣
   - 運動歴、運動頻度、使える器具
   - 怪我・痛み、苦手な運動、好きな運動
   - 改善したい習慣、トレーナーメモ
5. 保存すると顧客一覧に表示されます。
6. 顧客カードから編集・削除できます。

必須項目は顧客名と目標です。数値項目に不正な値を入れた場合は、日本語のバリデーションエラーを表示します。

## 課題プラン作成の使い方

1. ナビゲーションから「課題作成」を開きます。
2. 登録済み顧客を選択します。
3. 課題タイトル、開始日、終了日、次回パーソナルトレーニング日を設定します。
4. 課題日数は日付範囲から自動計算されます。必要に応じて `＋` / `−` や「生成」で調整します。
5. 各日の実施場所、休養日、トレーニング、有酸素、ケア、食事、睡眠、体重記録、水分、習慣、学習、メモ、チェック項目を編集します。
6. AI生成またはモック生成で課題案を作成できます。
7. 生成後も各入力欄から自由に手直しできます。
8. 「課題プランを保存」を押すとlocalStorageに保存されます。
9. 保存済み課題プランは右側の一覧から再読み込みできます。

## 書籍学習課題の使い方

### 書籍とセクションを登録する

1. ナビゲーションから「書籍管理」を開きます。
2. 書籍名、著者名、メモを入力して書籍を登録します。
3. 登録した書籍を選択します。
4. 章、ラベル、ページ範囲、表示順を入力して学習セクションを追加します。

書籍本文は保存しません。PDFにも本文は転載せず、書籍名・章・ラベル・ページ範囲だけを表示します。

### 4日進む・2日復習で割り当てる

1. `/assignments` を開きます。
2. 「書籍学習課題の自動割り当て」で書籍を選択します。
3. 「進む日数」に `4`、「復習日数」に `2` を入力します。
4. 休養日に学習を入れたくない場合は「休養日は飛ばす」を有効にします。
5. 「学習課題を割り当て」を押します。
6. 各日の学習課題に、登録した書籍セクションの範囲が入ります。
7. 割り当て後も手動で修正できます。

## PDF出力の使い方

1. `/assignments` で顧客と課題プランを作成します。
2. 顧客が選択されていると、PDFプレビューが表示されます。
3. 内容を確認します。
4. 「PDFをダウンロード」を押すとPDFを保存できます。

PDFはA4縦です。スマホで見たときに「今日やること」が分かりやすいよう、日別カード、短い要約、大きめのチェック欄を優先しています。

PDF出力モードは以下の2種類です。

- 基本1枚: 情報量を絞って1ページにまとめる想定
- 読みやすさ優先で2枚まで許可: 日別カードを最大2ページに分割して表示

## データ保存について

このMVPではバックエンドDBを使わず、ブラウザの `localStorage` に保存します。

保存対象は以下です。

- 顧客情報
- 課題プラン
- 書籍・学習セクション
- テンプレート

保存データは `src/lib/storage` のlocalStorage保存層で読み書きし、`src/lib/validation` のzod schemaで検証します。SSR中やlocalStorageが使えない環境では空配列などのフォールバック値を返します。

注意点:

- 同じブラウザ・同じ端末では再読み込み後もデータが残ります。
- 別端末・別ブラウザには同期されません。
- ブラウザのデータ削除、シークレットモード終了、ストレージ制限などで消える可能性があります。
- 実運用では認証、DB保存、バックアップ、権限管理を追加してください。

## 注意事項

- このアプリは医療診断や治療を目的としません。
- 怪我や痛みがある場合は無理をせず、痛みが出たら運動を中止してください。
- 不安がある場合や症状が続く場合は、トレーナーや適切な専門家に相談してください。
- AI生成結果はあくまで課題案です。トレーナーが顧客の状態を確認し、必要に応じて修正してください。
- 書籍本文はPDFに転載しないでください。著作権保護のため、書籍名・章・ページ範囲などの参照情報だけを扱ってください。
- 顧客情報を扱うため、実運用では認証、DB管理、通信の保護、権限管理、監査ログなどを追加してください。
- localStorage保存はMVP向けです。共有端末や本番運用では、顧客情報の漏えいに注意してください。
